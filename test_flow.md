# 聊天记录功能测试流程

## 修改总结

### 1. 移除了通讯录卡片上的同步按钮
- 不再在每个通讯录卡片上显示"同步聊天记录"按钮
- 避免了界面拥挤，保持简洁

### 2. 在聊天记录标题处添加了操作按钮
- **刷新按钮**: 刷新当前联系人的聊天记录
- **同步按钮**: 同步当前联系人的聊天记录到本地，然后刷新显示
- 两个按钮都只在选中联系人时启用

### 3. 修复了点击通讯录不加载聊天记录的问题
- 修改了通讯录点击动作，直接调用ajax查询接口
- 不再依赖reload-init-action和enable条件
- 确保点击通讯录时立即发送请求加载聊天记录

### 4. 修改了所有相关动作
- 通讯录点击动作
- 刷新按钮动作
- 同步按钮动作
- 空状态提示中的同步链接动作

## 使用流程

### 正常流程：
1. **选择员工** → 左侧员工列表点击员工卡片
2. **查看通讯录** → 中间区域显示该员工的通讯录
3. **选择联系人** → 点击通讯录卡片选中联系人
   - ✅ 自动发送请求：`hrm.travel_chat_record_list`
   - ✅ 右侧显示聊天记录
4. **刷新记录** → 点击"刷新"按钮重新加载聊天记录
5. **同步记录** → 点击"同步"按钮同步数据到本地并刷新显示

### 空状态流程：
1. **选择联系人**后无聊天记录
2. 显示提示："暂无聊天记录，点击同步聊天记录获取数据"
3. 点击"同步聊天记录"链接
   - ✅ 调用 `hrm.sync_chat_record` 同步数据
   - ✅ 调用 `hrm.travel_chat_record_list` 查询数据
   - ✅ 刷新显示聊天记录

## 接口调用

### 1. 聊天记录查询接口
```javascript
// 调用时机：点击通讯录、点击刷新按钮、同步完成后
{
  tag: "ajax",
  api: "post:/template_data/data?service=hrm.travel_chat_record_list",
  data: {
    agency_id: "${selectedEmployee.agency_id}",
    employee_id: "${selectedEmployee.employee_id}", 
    contact_id: "${selectedContact.contact_id}"
  },
  adapt: {
    chatDetailList: "${data}"
  }
}
```

### 2. 聊天记录同步接口
```javascript
// 调用时机：点击同步按钮、点击空状态同步链接
{
  tag: "ajax",
  api: "post:/template_data/data?service=hrm.sync_chat_record",
  data: {
    agency_id: "${currentAgency.agency_id}",
    employee_id: "${selectedEmployee.employee_id}",
    contact_id: "${selectedContact.contact_id}"
  }
}
// 同步完成后自动调用查询接口刷新显示
```

## 界面状态

### 1. 未选中联系人
- 聊天记录区域显示："请点击左侧联系人查看聊天记录"
- 刷新/同步按钮禁用

### 2. 选中联系人但无聊天记录
- 聊天记录区域显示空状态提示
- 包含可点击的"同步聊天记录"链接
- 刷新/同步按钮启用

### 3. 选中联系人且有聊天记录
- 正常显示聊天记录列表
- 刷新/同步按钮启用

## 注意事项

1. **字段映射**: SQL查询已做字段别名映射，确保前后端字段兼容
2. **错误处理**: 前端有基本错误提示，后端需返回清晰错误信息
3. **性能**: 聊天记录可能较多，后端需做好分页或限制
4. `is_self`字段: 当前固定为0，需根据业务逻辑在前端计算
5. `owner_alias`字段: 当前为null，可能需要关联其他表查询
